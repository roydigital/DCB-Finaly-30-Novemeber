Mobile Back Button Functionality Test

Test Description

This test simulates the mobile back button functionality for the "Your Order" section (cart sidebar) in the DCB application.

Expected Behavior: When the cart is open on mobile devices, pressing the browser's back button should close the cart and return to the main menu.

Implementation Details

Implemented Features:

  • History state management when opening/closing cart on mobile
  • popstate event listener to handle back button presses
  • Prevents double-closing with state tracking
  • Only activates on mobile viewports (< 768px)

Code Implementation

The following JavaScript code has been added to index.html:


// In DOMContentLoaded event listener:
window.addEventListener('popstate', (event) => {
    const sidebar = document.getElementById('cart-sidebar');
    // Check if we're handling a cart state change
    if (event.state && event.state.cartOpen === false) {
        // This is a state we pushed when closing cart, ignore it
        return;
    }
    if (window.innerWidth < 768 && sidebar && !sidebar.classList.contains('hidden')) {
        // Cart is open and back button was pressed - close it
        sidebar.classList.add('translate-x-full');
        setTimeout(() => sidebar.classList.add('hidden'), 300);
    }
});

// In toggleCart function (mobile section):
if (window.innerWidth < 768) {
    if (sidebar.classList.contains('hidden')) {
        // Opening cart
        history.pushState({ cartOpen: true }, '', '#cart');
    } else {
        // Closing cart  
        history.pushState({ cartOpen: false }, '', window.location.pathname);
    }
}
